開發流程

這份文件內容說明我們每次 開發/修改 網頁 介面/功能 時須共同遵守的流程以及規範，以及一些開發輔助工具的使用方法

話說在前

此文件內容有可能隨時更動，建議每次對網頁或程式動手前先來看一下，以免做白工

此流程假設開發者...

  • 的開發環境為 Linux 或 OSX
    • 明明都是mac><
    • 我有 linux 啊
    • 桌機喔？
  • 使用 html 及 javascript 開發網頁功能
  • 使用 github 幫助版本控管

規範流程是為了增進團隊效率，儘管如此也不該變成個人工作障礙，

若有個人的開發習慣且不影響共同開發效率，ex:我習慣在 windows 用記事本寫 code 而且神速

請用 comment 的方式提出，討論之後會再修改流程

傻瓜流程

  1. 下載最新版：git pull 更新本機的 source 到最新版
  2. 修改：Debug、加新功能...
  3. 測試：使用本機的網頁瀏覽器測試
  4. 部署：git push 上傳新版本
